# CommandHub V1.2 测试用例

## 文档目的

这份文档用于沉淀 V1.2 的核心测试场景，统一说明输入、验证点和当前覆盖方式。

- 自动化：已由 SwiftPM XCTest 覆盖
- 手工：当前仓库暂无 UI 自动化，需要人工确认

## 覆盖总览

| 编号 | 场景 | 重点验证 | 覆盖方式 |
| --- | --- | --- | --- |
| TC-01 | 基础命令 | parser 基本识别 | 自动化 |
| TC-02 | 多行 + 混合文本 | 过滤注释与噪音、拆分多命令 | 自动化 |
| TC-03 | 复杂 shell 命令 | `&&`、`|`、管道不截断 | 自动化 |
| TC-04 | `sudo` 前缀 | 不依赖关键词开头 | 自动化 |
| TC-05 | Git 高频命令 | 常见 Git 命令收录 | 自动化 |
| TC-06 | K8s / 运维命令 | 常见 `kubectl` 命令收录 | 自动化 |
| TC-07 | Docker 命令 | 常见 Docker 命令收录 | 自动化 |
| TC-08 | 错误 / 噪音输入 | parser 过滤能力 | 自动化 |
| TC-09 | 重复测试 | SQLite 精确去重 | 自动化 |
| TC-10 | 模糊搜索 | `kctl/gco/gcm/dps` 命中 | 自动化 |
| TC-11 | 排序测试 | 空 query 下 usage 排序 | 自动化 |
| TC-12 | 最近使用测试 | recency 对排序的影响 | 自动化 |
| TC-13 | 超长命令 | 存储完整、UI 截断 | 自动化 + 手工 |
| TC-14 | 特殊字符 | parser / SQLite 稳定性 | 自动化 |
| TC-15 | 大量数据压测 | 候选集限制与搜索稳定性 | 自动化 |

## 详细用例

## TC-01 基础命令

**输入**

```bash
ls -la
pwd
cd /var/log
echo "hello world"
cat /etc/hosts
```

**预期**

- 5 条命令全部被识别
- 顺序与输入一致
- 不发生错误过滤

## TC-02 多行 + 混合文本

**输入**

```bash
# 查看 pod
kubectl get pods

# 查看日志
kubectl logs my-pod

some random text

docker ps
```

**预期**

- `#` 注释行被过滤
- `some random text` 被忽略
- 返回 3 条命令：`kubectl get pods`、`kubectl logs my-pod`、`docker ps`

## TC-03 复杂 shell 命令

**输入**

```bash
cd project && npm run dev
git checkout main && git pull
kubectl get pods | grep api
cat file.txt | awk '{print $1}'
```

**预期**

- `&&` 和 `|` 不会导致错误截断
- 每一行作为完整命令保存

## TC-04 `sudo` 前缀

**输入**

```bash
sudo kubectl get pods
sudo apt update
sudo systemctl restart nginx
```

**预期**

- 3 条命令全部识别
- parser 不要求命令必须从 `git` / `kubectl` / `docker` 等关键词开头

## TC-05 Git 高频命令

**输入**

```bash
git status
git checkout -b feature/test
git commit -m "fix bug"
git push origin main
git pull --rebase
git log --oneline
```

**预期**

- 6 条命令全部保存
- 后续可用于模糊搜索和 usage 排序

## TC-06 K8s / 运维命令

**输入**

```bash
kubectl get pods -A
kubectl describe pod my-pod
kubectl logs my-pod -f
kubectl exec -it my-pod -- /bin/bash
kubectl delete pod my-pod
```

**预期**

- 5 条命令全部识别
- 带参数、交互参数和 `--` 的写法不出错

## TC-07 Docker 命令

**输入**

```bash
docker ps
docker images
docker exec -it container /bin/bash
docker build -t my-app .
docker run -d -p 8080:80 nginx
```

**预期**

- 5 条命令全部识别
- 路径、端口映射、标签参数不会导致误判

## TC-08 错误 / 噪音输入

**输入**

```text
hello world
this is not command
123456
a
#
```

**预期**

- 不保存任何命令
- 不污染 SQLite 历史

## TC-09 重复测试

**输入**

```bash
git status
git status
git status
```

**预期**

- 仅保存 1 条 `git status`
- SQLite 仍保持精确去重

## TC-10 模糊搜索

**查询**

```text
kctl
gco
gcm
dps
```

**预期**

- `kctl` 命中 `kubectl get pods`
- `gco` 命中 `git checkout`
- `gcm` 命中 `git commit`
- `dps` 命中 `docker ps`

## TC-11 排序测试

**前置数据**

```text
git status        使用 10 次
kubectl get pods  使用 3 次
docker ps         使用 1 次
```

**操作**

- 搜索空 query

**预期顺序**

1. `git status`
2. `kubectl get pods`
3. `docker ps`

## TC-12 最近使用测试

**前置数据**

- `git status`：更早使用
- `kubectl get pods`：刚使用
- 二者 usage 相同

**操作**

- 搜索空 query

**预期**

- `kubectl get pods` 排在 `git status` 前面

## TC-13 超长命令

**输入**

```bash
kubectl get pods -n default --field-selector=status.phase=Running -o jsonpath='{.items[*].metadata.name}'
```

**预期**

- parser 可完整识别
- SQLite 可完整存储
- UI 列表显示应可读，不应导致布局异常

**备注**

- 当前自动化只覆盖“解析和存储完整”
- UI 截断策略仍需手工验证

## TC-14 特殊字符

**输入**

```bash
echo $PATH
echo `date`
echo "$(whoami)"
grep "error" log.txt
```

**预期**

- parser 不崩溃
- SQLite 可正常保存
- 搜索链路不因特殊字符报错

## TC-15 大量数据压测

**输入生成**

```bash
for i in {1..500}; do echo "git test-$i"; done
```

**预期**

- 500 条命令可写入
- 搜索不返回全量历史
- 候选集限制继续生效
- 搜索过程无明显卡顿

## 自动化测试映射

- parser 相关：`Tests/CommandHubTests/CommandParserTests.swift`
- 搜索 / 排序 / 最近使用 / 大量数据：`Tests/CommandHubTests/SearchServiceTests.swift`
- 去重 / 存储：`Tests/CommandHubTests/StorageServiceTests.swift`

## 当前缺口

- UI 级“超长命令显示截断”尚未接入自动化
- 若后续增加 SwiftUI UI test，可将 TC-13 从“自动化 + 手工”升级为纯自动化
