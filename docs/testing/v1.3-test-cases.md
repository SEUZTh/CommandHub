# CommandHub V1.3 测试用例

## 文档目的

这份文档用于沉淀 V1.3 的核心测试场景，统一说明输入、验证点和当前覆盖方式。

- 自动化：已由 SwiftPM XCTest 覆盖
- 手工：当前仓库暂无 UI 自动化，浏览器权限与真实前台应用切换需要人工确认

## 覆盖总览

| 编号 | 场景 | 重点验证 | 覆盖方式 |
| --- | --- | --- | --- |
| TC-01 | 基础命令解析 | parser 基本识别能力 | 自动化 |

| TC-03 | 复杂 shell 命令 | `&&`、管道、引号不截断 | 自动化 |
| TC-04 | SQLite 持久化 | 命令写入本地数据库 | 自动化 |

| TC-06 | 同 context 重复捕获 | 只更新 `capture_count / last_seen_at / url` | 自动化 |

| TC-20 | 大量数据搜索 | 候选集限制与性能稳定性 | 自动化 |

## 详细用例

## TC-01 基础命令解析

**输入**

```bash
ls -la
pwd
cd /var/log
echo "hello world"
cat /etc/hosts
```

**预期**

- 5 条命令全部被识别
- 顺序与输入一致
- 不发生错误过滤

## TC-02 多行与混合文本

**输入**

```bash
# 查看 pod
kubectl get pods

# 查看日志
kubectl logs my-pod

some random text

docker ps
```

**预期**

- 注释行被过滤
- 噪音文本被忽略
- 返回 3 条命令

## TC-03 复杂 shell 命令

**输入**

```bash
cd project && npm run dev
git checkout main && git pull
kubectl get pods | grep api
cat file.txt | awk '{print $1}'
```

**预期**

- 每一行都作为完整命令保存
- `&&`、`|`、引号不导致错误截断

## TC-04 SQLite 持久化

**操作**

- 保存 `git status`
- 重建存储服务实例

**预期**

- 命令仍可从 SQLite 读出
- 不依赖内存态历史

## TC-05 同命令多环境

**输入**

- `kubectl get pods` in `https://prod.example.com/pods`
- `kubectl get pods` in `https://dev.example.com/pods`

**预期**

- `commands` 中只有 1 条 `kubectl get pods`
- `command_contexts` 中有 2 条 context
- 两条 context 的 `env` 分别为 `prod`、`dev`

## TC-06 同 context 重复捕获

**输入**

- `kubectl get pods` in `https://prod.example.com/a`
- `kubectl get pods` in `https://prod.example.com/b`

**预期**

- 只保留 1 条 context row
- `capture_count = 2`
- `last_seen_at` 更新
- `url` 更新为最近一次观察值
- `usage_count` 和 `last_used_at` 不变化

## TC-07 `markUsed` 命中优先

**前置条件**

- 同一命令已有 `prod` 和 `dev` 两条 context
- 搜索命中的是 `prod` context

**操作**

- 选择该命令
- 同时传入 `matchedContextID = prod`，`fallbackContext = dev`

**预期**

- 全局 `usage_count` 增加
- `prod` context 的 `usage_count` 增加
- `dev` context 的 `usage_count` 不增加

## TC-08 usage-only context

**前置条件**

- 命令存在，但没有任何 context row

**操作**

- `markUsed(commandID, matchedContextID: nil, fallbackContext: prod)`

**预期**

- 新增 1 条 context row
- `capture_count = 0`
- `usage_count = 1`
- `last_seen_at = nil`
- `last_used_at` 被写入

## TC-09 Env 识别规则

**输入与预期**

- `https://prod.example.com/dashboard?env=stg` -> `staging`
- `https://api-prod-eu.example.com/pods` -> `prod`
- `https://example.com/env/prod/dashboard` -> `prod`

**重点**

- query `env` 优先级最高
- subdomain token 能被识别
- host 无信号时可回落到 path

## TC-10 Env 误判保护

**输入**

```text
https://product.example.com/dashboard
```

**预期**

- 不识别为 `prod`
- 只允许完整 token 命中 env 映射

## TC-11 缩写模糊搜索

**查询**

```text
kctl
gco
```

**预期**

- `kctl` 命中 `kubectl`
- `gco` 命中 `git checkout`

## TC-12 domain 优先排序

**前置数据**

- 当前 context：`prod.cluster-a.example.com (prod)`
- 命令 A：domain 完全匹配
- 命令 B：env 一致但 domain 不同

**预期**

- 命令 A 排在命令 B 前面
- `matchedContext` 为 domain 精确命中的 context

## TC-13 `Current Env Only`

**前置数据**

- 当前 context：`prod.a.example.com (prod)`
- 命令 A：`prod.a.example.com`
- 命令 B：`prod.b.example.com`
- 命令 C：`dev.example.com`

**操作**

- scope 切换到 `Current Env Only`

**预期**

- 仅返回命令 A
- 过滤优先级固定为 `domain > env`
- 不接受“任一命中即可”的宽松逻辑

## TC-14 无当前 context 展示

**前置数据**

- 同一命令存在多条 context
- 其中一条 `usage_count` 更高，或在同 usage 下 `last_used_at` 更新

**预期**

- 当前无 context 时，`displayContext` 选择顺序为：
  - `usage_count DESC`
  - `last_used_at DESC`
  - `last_seen_at DESC`

## TC-15 Launcher 激活

**操作**

- 打开 Launcher

**预期**

- `query` 被清空
- `scope` 重置为 `All`
- 重新解析当前 context
- 结果列表重新加载
- 默认选中第一项

## TC-16 无 web context 状态

**前置条件**

- 当前前台应用为 `iTerm2`

**预期**

- 顶部状态为 `Current: iTerm2, no web context`
- `Current Env Only` 不可用
- 搜索仍能工作

## TC-17 浏览器权限拒绝

**手工步骤**

1. 前台切到 Chrome 或 Safari
2. 拒绝 CommandHub 的自动化权限
3. 复制一条命令并打开 Launcher

**预期**

- 命令仍会保存
- 搜索仍可用
- 顶部状态显示 `Browser context permission required`
- 不因拒绝权限崩溃或阻塞

## TC-18 浏览器真实 URL 采集

**手工步骤**

1. 前台切到 Chrome
2. 打开 `https://prod.example.com`
3. 复制一条命令
4. 再切到 Safari，打开 `https://dev.example.com`
5. 复制同一条命令

**预期**

- 两次复制生成两个不同 context
- `sourceApp` 分别记录为对应浏览器
- 后续搜索可根据当前环境重排

## TC-19 上下文标签展示

**手工步骤**

- 打开 Launcher，确保结果中存在带 context 的命令

**预期**

- 顶部状态栏显示当前 context 文案
- 结果 row 正常展示 `[env] [domain] [sourceApp]`
- 长文本不导致布局异常

## TC-20 大量数据搜索

**输入生成**

```bash
for i in {1..500}; do echo "git test-$i"; done
```

**预期**

- 大量命令可写入
- 搜索默认只取候选集，不扫描并展示全量历史
- 搜索过程无明显卡顿
