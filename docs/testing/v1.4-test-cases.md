# CommandHub V1.4 测试用例

## 文档目的

这份文档用于沉淀 V1.4 的新增测试场景，覆盖知识库化后的数据、搜索、交互和管理能力。

- 自动化：SwiftPM XCTest
- 手工：真实 macOS UI、菜单行为、窗口布局与快捷键

## 覆盖总览

| 编号 | 场景 | 重点验证 | 覆盖方式 |
| --- | --- | --- | --- |

## 详细用例

## TC-01 旧库迁移

**前置条件**

- 准备 V1.3 数据库，仅包含 `commands` 和 `command_contexts`

**预期**

- 启动后补齐：
  - `category`
  - `is_favorite`
  - `alias`
  - `note`
  - `workspace_id`
  - `workspaces` 表
- 原有命令与 context 数据仍可读取
- 旧命令会按规则回填 `category`

## TC-02 分类规则

**输入**

```text
git status
docker ps
kubectl get pods
pnpm install
ssh root@host
sudo docker images
custom-tool run
```

**预期**

- 分类分别为：
  - `Git`
  - `Docker`
  - `Kubernetes`
  - `Node`
  - `Remote`
  - `Docker`
  - `General`

## TC-03 alias / note 归一

**操作**

- alias 保存 `"  Pods  "`
- note 保存 `"   "`

**预期**

- alias 实际落库为 `Pods`
- note 实际落库为 `NULL`

## TC-04 favorite 切换

**操作**

- 对同一条命令执行收藏与取消收藏

**预期**

- `is_favorite` 在 `0 / 1` 间切换
- Launcher 行展示星标同步更新

## TC-05 workspace 生命周期

**操作**

- 新建 `"  Project A  "`
- 再新建 `"Project A"`
- 重命名为 `"  Project B "`

**预期**

- 首次创建成功，名称保存为 `Project A`
- 第二次创建因重名失败
- 重命名后名称保存为 `Project B`

## TC-06 workspace 解绑

**前置条件**

- 命令已绑定某个 workspace

**操作**

- 删除该 workspace

**预期**

- workspace 被删除
- 命令仍存在
- 命令的 `workspace_id` 变为 `NULL`

## TC-07 alias 搜索

**前置条件**

- 命令：`kubectl get namespaces`
- alias：`pods`

**查询**

```text
pods
```

**预期**

- 能召回原命令
- 结果第一行展示 alias，第二行仍显示原始命令

## TC-08 note 弱匹配

**前置条件**

- 命令：`git status`
- note：`production cluster checklist`

**查询**

```text
production
```

**预期**

- V1.4 不保证仅靠 note 关键词被稳定召回
- note 仅在已有候选集中参与弱加分

## TC-09 favorite 排序约束

**前置条件**

- 命令 A：普通命令，文本强匹配
- 命令 B：favorite 命令，文本无关

**预期**

- B 不能仅靠 favorite 进入结果
- 在文本都命中的前提下，favorite 可提升排序

## TC-10 二级过滤

**过滤组合**

- `favoritesOnly = true`
- `category = Git`
- `workspace = Platform`

**预期**

- 仅返回同时满足三者的命令

## TC-11 selection 保留

**前置条件**

- 当前已选中某条命令
- 刷新后该命令仍在结果中

**预期**

- selection 保持不变
- 详情区不跳项

## TC-12 selection 迁移

**前置条件**

- 当前已选中某条命令
- 刷新后该命令消失，但结果列表仍非空

**预期**

- 优先选中原位置后的相邻项
- 若后项不存在则退到前项

## TC-13 Launcher 双栏

**操作**

- 在正常窗口宽度下打开 Launcher

**预期**

- 左侧显示结果列表
- 右侧显示详情区

## TC-14 窄窗口降级

**操作**

- 缩窄 Launcher 窗口

**预期**

- 布局自动切换为上下结构
- 结果与详情仍可读

## TC-15 详情区编辑

**操作**

- 在详情区编辑 alias、note、workspace

**预期**

- 编辑后结果与详情同步刷新
- 不会把旧草稿误写到新选中项

## TC-16 单击改为选中

**操作**

- 鼠标单击结果行

**预期**

- 仅改变选中项
- 不会立即复制命令

## TC-17 快捷键

**验证**

- `Cmd + Shift + F`：收藏 / 取消收藏
- `Cmd + E`：聚焦 alias 输入框
- `Cmd + Enter`：复制并关闭窗口
- `Cmd + F`：聚焦搜索框

## TC-18 Settings 管理页

**操作**

- 通过 `CommandHub > Settings...` 打开设置页

**预期**

- 可新增 / 重命名 / 删除 workspace
- 列表中展示每个 workspace 的当前命令数量
