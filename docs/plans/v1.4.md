# CommandHub V1.4 计划

## 摘要

- V1.4 将 CommandHub 从“上下文感知命令历史”升级为“个人命令知识库”。
- 核心方向不再是继续扩展采集源，而是增强命令条目的可组织性、可编辑性和可管理性。
- 本版围绕四件事展开：
  - 命令元数据：`category / favorite / alias / note / workspace`
  - Launcher 双栏结果 + 详情视图
  - 搜索过滤与排序升级
  - Settings 中的 workspace 管理

## 关键变更

### 数据模型与迁移

- `commands` 新增列：
  - `category TEXT`
  - `is_favorite INTEGER NOT NULL DEFAULT 0`
  - `alias TEXT`
  - `note TEXT`
  - `workspace_id TEXT`
- 新增 `workspaces` 表：
  - `id TEXT PRIMARY KEY`
  - `name TEXT NOT NULL UNIQUE`
  - `created_at INTEGER NOT NULL`
- 数据库升级采用增量迁移：
  - 启动时检查列/表/索引是否存在，不依赖一次性重建
  - 仅对 `category IS NULL` 的旧命令执行回填
  - 不覆盖已有非空 `category`
- 规范化规则：
  - `alias / note` 写入前 `trim`
  - `trim` 后为空时写入 `NULL`
  - workspace 名称写入前 `trim`
  - workspace 名称不允许为空或重名
- 删除规则：
  - 删除命令时同步删除关联 `command_contexts`
  - 删除 workspace 时仅清空命令的 `workspace_id`

### 分类、搜索与排序

- 新增统一分类解析器 `CommandCategoryResolver`，保存命令和迁移回填共用同一规则。
- 分类规则按首个可执行 token 判断，并忽略前导 `sudo`：
  - `git` -> `Git`
  - `docker` -> `Docker`
  - `kubectl` -> `Kubernetes`
  - `npm / yarn / pnpm` -> `Node`
  - `ssh / scp` -> `Remote`
  - 其他 -> `General`
- 新增 `SearchFilters`：
  - `favoritesOnly`
  - `category`
  - `workspaceID`
- SQL 粗筛仅匹配：
  - `command`
  - `alias`
- `note` 不参与 SQL 粗筛，只在内存评分阶段做弱匹配。
- 最终排序引入 `favoriteBonus`，但遵守两个约束：
  - `favorite` 只增强排序，不扩大召回集合
  - `textMatch == 0` 时不能仅靠 `favorite` 进入结果

### Launcher 交互

- Launcher 升级为双栏：
  - 左侧结果列表
  - 右侧详情区
- 当窗口宽度不足时自动降级为上下布局，保持可读性。
- 结果行展示规则：
  - 第一行：`alias ?? command`
  - 第二行：原始命令 + `category / env / domain / sourceApp` chips
  - 右侧：`favorite` 与 `usageCount`
- 单击行为从“立即复制”改为“仅选中”，主复制路径调整为：
  - `Enter`：复制
  - `Cmd + Enter`：复制并关闭
- 详情区支持内联编辑：
  - `favorite`
  - `alias`
  - `note`
  - `workspace`
- 新增快捷键：
  - `Cmd + F`：聚焦搜索框
  - `Cmd + Shift + F`：收藏 / 取消收藏
  - `Cmd + E`：聚焦 alias 输入框
  - `Backspace` 或 `Cmd + Delete`：删除命令（弹确认）

### Settings 页面

- Settings 不再是占位文本，V1.4 起用于 workspace 管理。
- 当前 Settings 支持：
  - 新建 workspace
  - 重命名 workspace
  - 删除 workspace
  - 查看每个 workspace 当前绑定的命令数量

## 对外行为

- CommandHub 现在不仅能“找回命令”，也能把命令整理成稳定的知识条目。
- 用户可以通过收藏、别名、备注和 workspace 对命令进行长期管理。
- 搜索结果支持按 `Favorites / Category / Workspace` 过滤。
- Settings 页面可通过应用菜单中的 `Settings...` 打开。

## 测试计划

- 存储与迁移：
  - V1.3 数据库升级后字段完整、旧数据不丢失
  - category 规则回填正确
  - alias/note 空白值归一为 `NULL`
  - workspace 名称会被 `trim` 且拒绝重名
- 搜索：
  - alias 命中能召回
  - note 只做弱加分，不扩大 SQL 召回
  - favorite 只能增强排序，不改变召回集合
  - `favorites/category/workspace` 过滤正确
- ViewModel：
  - 激活 Launcher 时加载 workspace
  - 编辑元数据后即时刷新
  - selection 在刷新后按既定规则保留或迁移
- 手工：
  - 双栏 / 上下布局切换
  - 详情区编辑体验
  - Settings 中 workspace 管理

## 假设与默认值

- V1.4 不做 AI 自动 alias/note、自动 workspace 推断、FTS 升级和多设备同步。
- `category` 为系统规则字段，V1.4 不提供手工编辑。
- `note` 当前主要承担辅助说明和弱排序作用，不作为强召回字段。
- 保持 `matchedContext / displayContext` 语义不变，不改写 `command_contexts` 结构。
