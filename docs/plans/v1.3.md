# CommandHub V1.3 计划

## 摘要

- V1.3 将 CommandHub 从“命令历史搜索”升级为“上下文感知命令系统”。
- `commands` 继续保持命令全局唯一，新增 `command_contexts` 表记录同一命令在不同环境下的独立捕获与使用统计。
- 搜索默认展示全部结果，但根据当前上下文重排；同时提供 `Current Env Only` 过滤模式，过滤优先级固定为 `domain > env`。
- 搜索结果与持久模型解耦：`matchedContext` 只存在于搜索结果态，不进入 `CommandItem` 本体。

## 关键变更

### 数据模型与持久化

- `commands` 表保留：
  - `id TEXT PRIMARY KEY`
  - `command TEXT NOT NULL UNIQUE`
  - `usage_count INTEGER NOT NULL DEFAULT 0`
  - `last_used_at INTEGER`
  - `created_at INTEGER NOT NULL`
- 新增 `command_contexts` 表：
  - `id TEXT PRIMARY KEY`
  - `command_id TEXT NOT NULL`
  - `context_key TEXT NOT NULL`
  - `domain TEXT`
  - `url TEXT`
  - `env TEXT`
  - `source_app TEXT`
  - `capture_count INTEGER NOT NULL DEFAULT 0`
  - `usage_count INTEGER NOT NULL DEFAULT 0`
  - `last_seen_at INTEGER`
  - `last_used_at INTEGER`
  - `created_at INTEGER NOT NULL`
- 约束与索引：
  - `UNIQUE(command_id, context_key)`
  - `INDEX command(command)`
  - `INDEX command_contexts(command_id)`
  - `INDEX command_contexts(env, domain, last_used_at)`
- `context_key` 固定为 `<source_app>|<normalized_domain>|<normalized_env>`：
  - `nil -> ""`
  - 全部转小写
  - `domain` 去端口、去尾部 `.`
  - 不允许写入 `"nil"` 或 `"null"`
- 重复捕获同一 `command_id + context_key` 时只更新：
  - `capture_count += 1`
  - `last_seen_at = now`
  - `url = latestObservedURL`
  - 不更新 `usage_count`、`last_used_at`
- `commands.source` 自 V1.3 起废弃：
  - 新代码不再读写
  - 不参与排序、过滤、展示
  - 仅为兼容历史库保留

### 类型与接口

- `CommandContext` 固定字段：
  - `url: String?`
  - `domain: String?`
  - `env: String?`
  - `sourceApp: String?`
- 新增 `CommandContextStat`，表示单个上下文 bucket 的统计信息。
- `CommandItem` 只表示持久数据本体：
  - `id`
  - `command`
  - `usageCount`
  - `lastUsedAt`
  - `createdAt`
  - `contexts: [CommandContextStat]`
- 新增 `SearchResultItem`：
  - `command: CommandItem`
  - `matchedContext: CommandContextStat?`
  - `displayContext: CommandContextStat?`
  - `score: Double`
- 搜索接口升级为：
  - `search(query: String, currentContext: CommandContext?, scope: SearchScope) -> [SearchResultItem]`
- 存储接口升级为：
  - `save(command: String, context: CommandContext?)`
  - `fetchCandidates(query: String, currentContext: CommandContext?, scope: SearchScope, limit: Int) -> [CommandItem]`
  - `markUsed(commandID: String, matchedContextID: String?, fallbackContext: CommandContext?)`
- `markUsed` 规则固定：
  - 优先更新 `matchedContextID`
  - 没有命中上下文时，再用 `fallbackContext` upsert context row
  - 两者都没有时，只更新命令全局统计

### Context 采集与环境解析

- Context 采集只支持：
  - `Google Chrome`
  - `Safari`
- 当前台应用是支持的浏览器时，CommandHub 会尝试通过 AppleScript 读取当前 tab URL。
- 权限策略为懒触发：
  - 仅在需要浏览器 URL context 时请求自动化权限
  - 拒绝权限不会阻断命令采集与搜索
- 非浏览器前台应用只记录 `sourceApp`，不记录 `url/domain/env`。
- Env 识别顺序固定：
  1. URL query 中显式 `env`
  2. host 左侧子域 token
  3. host 其余 token
  4. path 前两级 segment
- token 按 `.` `-` `_` 切分，仅完整 token 命中。
- Canonical env 集合：
  - `prod`
  - `staging`
  - `dev`
  - `qa`
  - `test`
  - `uat`
  - `local`
- domain 正规化为完整 host，而不是根域。

### 搜索与排序

- 候选粗筛采用双桶策略：
  - `All`：优先取 context 命中候选，再用全局候选补齐
  - `Current Env Only`：优先取 context 命中候选，最终结果再按 `domain > env` 严格过滤
  - 无当前上下文：只走全局候选
- 最终评分模型：
  - `textMatch * 0.40`
  - `globalUsage * 0.20`
  - `globalRecency * 0.10`
  - `contextAffinity * 0.20`
  - `contextBehavior * 0.10`
- `contextAffinity` 规则固定：
  - domain 精确匹配：`1.0`
  - 当前与记录 env 明确冲突：`0`
  - env 精确匹配：`0.5`
  - source app 一致：`0.15`
  - 其他：`0`
- `textMatch == 0` 时，context 不允许强行召回命令。
- 无当前 context 时，`displayContext` 选择顺序固定为：
  - `usage_count DESC`
  - `last_used_at DESC`
  - `last_seen_at DESC`

### UI 与交互

- Launcher 顶部新增 context 状态栏，展示当前上下文状态。
- 提供 `All` / `Current Env Only` 分段切换。
- 结果列表展示命令文本与上下文 chips：
  - `[env]`
  - `[domain]`
  - `[sourceApp]`
- 顶部状态文案覆盖四类场景：
  - `Current: prod.xxx.com (prod)`
  - `Current: Chrome, web context unavailable`
  - `Browser context permission required`
  - `Current: iTerm2, no web context`
- 保持原有交互：
  - 上下键切换
  - 回车复制
  - 点击复制
  - `Esc` 关闭

## 对外行为

- 命令现在保存在本地 SQLite，而不是内存。
- 相同命令在多个环境下不会重复插入 `commands`，但会生成多条上下文统计。
- 当前环境可用时，结果会优先展示更匹配的环境命令。
- 当前环境不可用时，搜索回退到“匹配度 + 使用频率 + 最近使用”。

## 测试计划

- 存储测试：
  - 同一命令在 `prod/dev` 下只保留一条 `commands`
  - 相同 `context_key` 重复捕获只更新 `capture_count/last_seen_at/url`
  - `markUsed` 优先更新命中的 context
  - 无命中 context 时，fallback context 会创建 usage-only 上下文记录
- 搜索测试：
  - `kctl -> kubectl`
  - domain 精确匹配优先于仅 env 匹配
  - `Current Env Only` 严格按 `domain > env` 过滤
  - 无当前 context 时展示默认最常用上下文
- Context 解析测试：
  - query 参数 env 优先
  - 子域 token 与 path segment 可识别 env
  - 避免 `product.xxx.com` 这类误判
- ViewModel 测试：
  - 激活 Launcher 时刷新 context、重置 query 和 scope
  - 选择命令时优先回写 `matchedContext`
  - 当前上下文不可过滤时，scope 自动回退为 `All`

## 假设与默认值

- V1.3 不做自定义 env 规则编辑界面。
- V1.3 不扩展到 Firefox、Arc 或终端 host 粒度。
- 完整 URL 会存入本地数据库，但 UI 默认只展示 `domain/env/sourceApp`。
- 浏览器权限缺失只影响 context-aware 能力，不影响主流程。
